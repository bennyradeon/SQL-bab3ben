Modul SQL Lanjutan

Fokus pembelajaran: SELECT, WHERE, AND/OR/NOT, LIKE, IN, BETWEEN, ORDER BY, LIMIT. Latihan menggunakan db_sekolah dan tugas menggunakan db_kuliah.

MySQL phpMyAdmin Latihan logika

Tujuan

Kompetensi

  • Menampilkan kolom yang tepat dengan SELECT.
  • Menyaring data dengan WHERE dan logika AND/OR/NOT.
  • Mencari teks memakai LIKE.
  • Memilih beberapa nilai memakai IN.
  • Memilih rentang memakai BETWEEN.
  • Merapikan dan membatasi hasil memakai ORDER BY dan LIMIT.

Catatan

Pada modul ini kita belum masuk JOIN dan fungsi agregat. Kita kuatkan dulu dasar lanjutan sesuai materi kelas.

Target minimal
Kalian mampu membuat query gabungan: WHERE + LIKE/IN/BETWEEN + ORDER BY + LIMIT.

Alur Berpikir

Ikuti urutan ini setiap membuat query agar hasil mudah dicek dan tidak membingungkan.

  1. Pilih kolom → SELECT
  2. Pilih tabel → FROM
  3. Tambah syarat → WHERE
  4. Urutkan hasil → ORDER BY
  5. Batasi hasil → LIMIT

Contoh query rapi

SELECT nis, nama
FROM siswa
WHERE alamat = 'Jakarta'
ORDER BY nama ASC
LIMIT 5;

Hal yang sering keliru

  • Teks wajib pakai petik satu: 'Jakarta'.
  • Nama kolom harus persis sesuai tabel.
  • ORDER BY + LIMIT dipakai jika ingin “Top data”.

CREATE DATABASE db_sekolah;

CREATE TABLE siswa (
    id_siswa INT AUTO_INCREMENT PRIMARY KEY,
    nis VARCHAR(20) UNIQUE,
    nama VARCHAR(100) NOT NULL,
    jenis_kelamin ENUM('L','P'),
    tanggal_lahir DATE,
    alamat TEXT,
    tahun_masuk YEAR
);

CREATE TABLE kelas (
    id_kelas INT AUTO_INCREMENT PRIMARY KEY,
    nama_kelas VARCHAR(20),
    wali_kelas VARCHAR(100)
);

CREATE TABLE siswa_kelas (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_siswa INT,
    id_kelas INT,
    tahun_ajaran VARCHAR(9),
    FOREIGN KEY (id_siswa) REFERENCES siswa(id_siswa),
    FOREIGN KEY (id_kelas) REFERENCES kelas(id_kelas)
);

CREATE TABLE mata_pelajaran (
    id_mapel INT AUTO_INCREMENT PRIMARY KEY,
    nama_mapel VARCHAR(100)
);

CREATE TABLE nilai (
    id_nilai INT AUTO_INCREMENT PRIMARY KEY,
    id_siswa INT,
    id_mapel INT,
    semester INT,
    nilai_angka DECIMAL(5,2),
    FOREIGN KEY (id_siswa) REFERENCES siswa(id_siswa),
    FOREIGN KEY (id_mapel) REFERENCES mata_pelajaran(id_mapel)
);

CREATE TABLE buku (
    id_buku INT AUTO_INCREMENT PRIMARY KEY,
    judul VARCHAR(150),
    penulis VARCHAR(100),
    tahun_terbit YEAR,
    stok INT
);

CREATE TABLE peminjaman (
    id_pinjam INT AUTO_INCREMENT PRIMARY KEY,

    FOREIGN KEY (id_siswa) REFERENCES siswa(id_siswa),

Materi Inti

1) SELECT dan FROM

Di tahap awal belajar, kalian boleh memakai SELECT * untuk melihat isi tabel secara utuh. Ini berguna untuk mengenali nama kolom dan memastikan data sudah masuk. Namun setelah itu, biasakan menampilkan kolom yang memang diperlukan saja. Alasannya sederhana: hasil query jadi lebih ringkas, mudah dibaca, dan enak dipakai untuk laporan.

SELECT * FROM siswa;

SELECT nis, nama, alamat FROM siswa;

2) WHERE

Kalau kalian tidak menulis WHERE, database akan menampilkan semua baris data. Saat data mulai banyak, ini membuat kita sulit fokus. Karena itu, gunakan WHERE ketika kalian hanya butuh data yang memenuhi syarat tertentu, misalnya tahun masuk tertentu atau alamat tertentu. Biasakan juga mengecek: teks harus pakai petik satu, sedangkan angka tidak perlu.

SELECT nis, nama
FROM siswa
WHERE tahun_masuk = 2023;

3) AND / OR / NOT

Di kasus nyata, syarat sering lebih dari satu. Di sini kalian memakai logika. Gunakan AND kalau semua syarat harus benar, gunakan OR kalau cukup salah satu syarat benar, dan gunakan NOT untuk pengecualian. Kalau hasil query kosong, jangan panik dulu—cek lagi apakah datanya memang ada atau syaratnya terlalu ketat.

SELECT nis, nama
FROM siswa
WHERE tahun_masuk = 2023 AND jenis_kelamin = 'P';

SELECT judul, stok
FROM buku
WHERE stok = 3 OR stok = 4;

SELECT id_pinjam, status
FROM peminjaman
WHERE NOT status = 'Kembali';

4) LIKE

Kadang kalian tidak ingat data persis, tetapi ingat sebagian kata. Untuk kondisi seperti itu, gunakan LIKE. Ingat aturan pola: 'A%' artinya dimulai huruf A, sedangkan '%Putri%' artinya mengandung kata Putri di bagian mana pun. Pastikan tanda persen % kalian letakkan sesuai tujuan pencariannya.

SELECT nis, nama
FROM siswa
WHERE nama LIKE 'A%';

SELECT nis, nama
FROM siswa
WHERE nama LIKE '%Putri%';

5) IN dan BETWEEN

Kalau kalian menulis OR berkali-kali, query akan panjang dan mudah salah. Untuk itu gunakan IN saat pilihan datanya berupa daftar. Gunakan BETWEEN saat kalian memfilter rentang angka atau tahun. Ingat: BETWEEN termasuk batas bawah dan batas atas.

SELECT nis, nama, alamat
FROM siswa
WHERE alamat IN ('Jakarta','Depok','Bogor');

SELECT judul, tahun_terbit
FROM buku
WHERE tahun_terbit BETWEEN 2019 AND 2022;

6) ORDER BY dan LIMIT

Setelah data sudah benar, rapikan tampilannya. ORDER BY membantu mengurutkan hasil agar mudah dibaca, misalnya A–Z atau nilai terbesar dulu. Kalau kalian hanya butuh beberapa data teratas, gunakan LIMIT. Biasanya LIMIT dipasangkan dengan ORDER BY supaya jelas ‘teratas’ itu berdasarkan apa.

SELECT judul, stok
FROM buku
ORDER BY stok DESC
LIMIT 3;

Tugas (db_kuliah)

db_kuliah

Kerjakan tugas ini di db_kuliah. Pastikan query rapi dan sesuai instruksi.

Tugas 1 — Administrasi Mahasiswa

  1. Tampilkan nim, nama, angkatan mahasiswa angkatan 2022, urutkan nama A–Z.
  2. Tampilkan mahasiswa perempuan angkatan 2022.
  3. Tampilkan mahasiswa yang namanya mengandung Putri atau Sari.

Tugas 2 — Mata Kuliah

  1. Tampilkan kode_mk, nama_mk, sks untuk sks = 3, urutkan nama_mk A–Z.
  2. Tampilkan mata kuliah dengan kode diawali IF (LIKE).
  3. Tampilkan mata kuliah untuk jurusan id 1, 2, 3 (IN).

Tugas 3 — KRS Sederhana

  1. Tampilkan KRS semester 1 saja.
  2. Tampilkan KRS semester 1 sampai 3 (BETWEEN).
  3. Tampilkan KRS dengan nilai A atau B (IN), urut semester ASC, batasi 10 baris.
Catatan
Gunakan ORDER BY dan LIMIT pada poin yang memerlukannya.

Tugas 4 — Kode Tantangan

Guru membagikan kode: [Fokus]-[Filter]-[Output]. Buat 1 query sesuai kode.

Fokus
  • M = mahasiswa
  • K = krs
  • MK = mata_kuliah
Filter
  • 1 = angkatan
  • 2 = semester
  • 3 = nilai
Output
  • A = ORDER BY
  • L = LIMIT
  • B = BETWEEN
  • I = IN
  • S = LIKE
Contoh kode: K-2-L
Fokus KRS, filter semester, hasil dibatasi dengan LIMIT.
