DB Sekolah — Peta Materi (Guru Menjelaskan)

Bayangkan database itu lemari arsip sekolah. Setiap tabel adalah laci. Kalau kita menulis query, itu seperti kita memberi instruksi: “Ambil data dari laci ini, pilih kertas yang ini, urutkan begini, tampilkan sekian.”

Laci data inti
siswa • kelas • buku
Laci penghubung
siswa_kelas • nilai • peminjaman
Skill yang paling sering
WHERE + JOIN + ORDER BY
Aturan emas membaca query (urut logika):
1) FROM ambil dari tabel mana, 2) JOIN gabungkan tabel lain (kalau perlu), 3) WHERE saring yang memenuhi syarat, 4) ORDER BY rapikan urutan, 5) LIMIT batasi jumlah hasil.
Struktur Database (SQL) sumber
Silakan buka dokumen berikut untuk CREATE DATABASE, CREATE TABLE, dan INSERT. Kamu bisa langsung copy-paste ke MySQL/MariaDB.
Buka dokumen struktur database
Catatan: setelah menjalankan skrip SQL pada dokumen tersebut, barulah query-query latihan di halaman ini bisa dicoba.
Kesalahan umum (biar cepat paham) hindari ini
  • Lupa WHERE → data kebanyakan (terlihat “benar” tapi tidak sesuai tugas).
  • JOIN tanpa ON → hasil meledak (cartesian product), datanya jadi tidak masuk akal.
  • LIMIT tanpa ORDER BY → hasil “acak” (karena DB tidak wajib mengurutkan data).
  • Salah kutip: string pakai '...', angka tidak pakai kutip.

Playground Query — DB Sekolah

Silakan klik: copy query, lalu buka keterangan guru untuk paham alasannya ✨

1) SELECT kolom penting (daftar kontak siswa)
SELECT ORDER BY
Tujuan: ambil data yang “cukup”, bukan semua kolom.
SELECT nis, nama, alamat
FROM siswa
ORDER BY nama ASC;
Penjelasan: Kita pilih kolom yang relevan saja. ORDER BY nama ASC membuat daftar rapi seperti absen kelas (A → Z).
Analogi: Kamu hanya fotokopi halaman yang dibutuhkan, bukan seluruh buku.
2) WHERE — filter siswa angkatan 2023
WHERE Filter 
SELECT nis, nama, tahun_masuk
FROM siswa
WHERE tahun_masuk = 2023;
Penjelasan: WHERE itu seperti “pilih siswa dengan syarat tertentu”.
Analogi: Dari semua murid di sekolah, kamu dipanggilkan hanya yang “angkatan 2023”.
3) AND — perempuan angkatan 2023
WHERE AND 
SELECT nis, nama, jenis_kelamin, tahun_masuk
FROM siswa
WHERE tahun_masuk = 2023 AND jenis_kelamin = 'P';
Penjelasan: AND artinya dua syarat wajib terpenuhi.
Analogi: “Masuk kelas” hanya untuk yang pakai seragam AND punya kartu pelajar.
4) LIKE — cari buku yang judulnya mengandung “Bahasa”
LIKE Pola teks 
SELECT id_buku, judul, penulis
FROM buku
WHERE judul LIKE '%Bahasa%';
Penjelasan: Tanda % = “boleh ada karakter apa saja”.
Analogi: Kamu mencari di katalog: “Yang penting ada kata Bahasa di judulnya, depan-belakang bebas.”
Catatan: Kalau “diawali Bahasa” pakai 'Bahasa%'. Kalau “diakhiri Bahasa” pakai '%Bahasa'.
5) IN — filter alamat tertentu
IN Daftar nilai 
SELECT nis, nama, alamat
FROM siswa
WHERE alamat IN ('Jakarta', 'Depok', 'Bogor');
Penjelasan: IN itu seperti “pilih dari daftar pilihan”.
Analogi: Dari daftar kota, kita centang Jakarta/Depok/Bogor saja.
6) BETWEEN — buku terbit 2019 s/d 2022
BETWEEN Rentang 
SELECT judul, tahun_terbit
FROM buku
WHERE tahun_terbit BETWEEN 2019 AND 2022;
Penjelasan: BETWEEN bersifat inklusif, artinya 2019 dan 2022 ikut.
Analogi: Seperti aturan “nilai 80–90”, maka 80 dan 90 juga dihitung.
7) ORDER BY + LIMIT — Top 3 stok terbanyak
ORDER BY LIMIT 
SELECT judul, stok
FROM buku
ORDER BY stok DESC
LIMIT 3;
Penjelasan: Kita urutkan dulu stok dari besar ke kecil, baru ambil 3 teratas.
Analogi: Seperti lomba lari: urutkan pemenang dulu, baru ambil juara 1–3.
8) JOIN — siswa + kelas + tahun ajaran (yang paling penting)
JOIN ORDER BY 
SELECT s.nis, s.nama, k.nama_kelas, sk.tahun_ajaran
FROM siswa s
JOIN siswa_kelas sk ON s.id_siswa = sk.id_siswa
JOIN kelas k ON sk.id_kelas = k.id_kelas
ORDER BY k.nama_kelas, s.nama;
Penjelasan langkah demi langkah:
1) FROM siswa s → mulai dari data siswa.
2) JOIN ke siswa_kelas → cari “baris penghubung” siapa siswa masuk kelas apa.
3) JOIN ke kelas → ambil nama kelasnya.

Analogi: Kamu punya daftar siswa dan daftar kelas. Nah, siswa_kelas itu seperti buku pembagian kelas yang mencatat: “Siswa A masuk kelas X”. Tanpa buku pembagian itu, kamu tidak bisa menempelkan siswa ke kelas yang tepat.
9) JOIN — peminjaman aktif (nama siswa + judul buku)
JOIN WHERE ORDER BY 
SELECT s.nama, b.judul, p.tanggal_pinjam, p.tanggal_kembali, p.status
FROM peminjaman p
JOIN siswa s ON p.id_siswa = s.id_siswa
JOIN buku b ON p.id_buku = b.id_buku
WHERE p.status = 'Dipinjam'
ORDER BY p.tanggal_pinjam DESC;
Penjelasan: Tabel peminjaman hanya menyimpan id_siswa & id_buku. JOIN membuat kita bisa menampilkan nama siswa dan judul buku.
Analogi: Struk pinjaman menyimpan “kode siswa & kode buku”; kita buka katalog untuk ubah kode menjadi nama nyata.
10) JOIN — laporan nilai semester 1
JOIN WHERE ORDER BY 
SELECT s.nama, m.nama_mapel, n.semester, n.nilai_angka
FROM nilai n
JOIN siswa s ON n.id_siswa = s.id_siswa
JOIN mata_pelajaran m ON n.id_mapel = m.id_mapel
WHERE n.semester = 1
ORDER BY s.nama, m.nama_mapel;
Penjelasan: Tabel nilai itu “jembatan” antara siswa dan mata pelajaran.
Analogi: Seperti rapor: ada nama siswa, ada mapel, ada nilai. Data mentahnya tersebar, JOIN menyatukannya jadi “rapor”.
Latihan cepat (DB Sekolah) 3 menit
  1. Tampilkan siswa perempuan yang alamat Jakarta.
  2. Tampilkan buku terbit setelah 2020 urut dari terbaru.
  3. Tampilkan peminjaman yang belum kembali beserta nama siswa dan judul buku.
Petunjuk: gunakan WHERE + AND, ORDER BY, dan JOIN.

DB Kuliah — Peta Materi (Guru Menjelaskan)

Di kampus, data itu seperti administrasi akademik: mahasiswa, jurusan, fakultas, dosen, mata kuliah, dan KRS. Tantangan utamanya biasanya JOIN bertingkat (mahasiswa → jurusan → fakultas) dan laporan KRS.

Laci data inti
mahasiswa • dosen • mata_kuliah
Laci penghubung
jurusan • fakultas • krs
Output populer
KRS + Nilai per semester
Analogi JOIN bertingkat:
Nama mahasiswa itu “orangnya”. Jurusan dan fakultas itu “alamat akademiknya”. Untuk menuliskan alamat lengkap, kamu harus melewati beberapa langkah: mahasiswa → jurusan → fakultas.
Struktur Database (SQL) sumber
Silakan buka dokumen berikut untuk CREATE DATABASE, CREATE TABLE, dan INSERT. Kamu bisa langsung copy-paste ke MySQL/MariaDB.
Buka dokumen struktur database
Catatan: setelah menjalankan skrip SQL pada dokumen tersebut, barulah query-query latihan di halaman ini bisa dicoba.
Kesalahan umum (biar tidak kepleset) cek dulu
  • Ingat: KRS menyimpan id_mahasiswa & id_mk, jadi untuk tampilkan nama harus JOIN.
  • Kalau ingin “dosen pengampu” → JOIN lagi dari mata_kuliah ke dosen.
  • Nilai KRS berupa char (A/B/C), jadi jangan dihitung sebagai angka.

Playground Query — DB Kuliah

Klik untuk copy dan buka penjelasan guru ✨

1) SELECT + ORDER BY (daftar mahasiswa rapi)
SELECT ORDER BY 
SELECT nim, nama, angkatan
FROM mahasiswa
ORDER BY angkatan DESC, nama ASC;
Penjelasan: Angkatan terbaru dulu (DESC), lalu nama A–Z (ASC).
Analogi: Data disusun seperti daftar mahasiswa per tahun masuk.
2) WHERE (angkatan 2022)
WHERE Filter 
SELECT nim, nama, angkatan
FROM mahasiswa
WHERE angkatan = 2022;
Penjelasan: Sama seperti menyaring daftar “angkatan 2022” saja.
3) LIKE (cari dosen mengandung “Lina”)
LIKE Pola teks 
SELECT nama_dosen, email
FROM dosen
WHERE nama_dosen LIKE '%Lina%';
Penjelasan: % itu wildcard. Yang penting “Lina” ada di mana pun.
4) IN (mahasiswa jurusan tertentu)
IN Daftar nilai 
SELECT nim, nama, id_jurusan
FROM mahasiswa
WHERE id_jurusan IN (1, 2, 3);
Penjelasan: IN lebih singkat daripada OR berkali-kali.
Analogi: Kamu memilih beberapa kelas sekaligus: “A, B, atau C”.
5) BETWEEN (angkatan 2021–2022)
BETWEEN Rentang 
SELECT nim, nama, angkatan
FROM mahasiswa
WHERE angkatan BETWEEN 2021 AND 2022;
Penjelasan: Rentang inklusif: 2021 & 2022 ikut.
6) ORDER BY + LIMIT (ambil 3 baris)
ORDER BY LIMIT 
SELECT nim, nama
FROM mahasiswa
ORDER BY nama ASC
LIMIT 3;
Penjelasan: LIMIT tanpa ORDER BY bisa “acak”. Maka kita rapikan dulu, baru ambil 3.
7) JOIN bertingkat (mahasiswa + jurusan + fakultas)
JOIN ORDER BY 
SELECT m.nim, m.nama, j.nama_jurusan, f.nama_fakultas, m.angkatan
FROM mahasiswa m
JOIN jurusan j ON m.id_jurusan = j.id_jurusan
JOIN fakultas f ON j.id_fakultas = f.id_fakultas
ORDER BY f.nama_fakultas, j.nama_jurusan, m.nama;
Langkah mudah:
1) Mulai dari mahasiswa,
2) sambungkan ke jurusan (pakai id_jurusan),
3) sambungkan jurusan ke fakultas (pakai id_fakultas).

Analogi: Menulis alamat lengkap: orangnya (mahasiswa) → RT/RW (jurusan) → kelurahan (fakultas). Bertahap, jangan lompat.
8) JOIN (mata kuliah + jurusan + dosen)
JOIN ORDER BY 
SELECT mk.kode_mk, mk.nama_mk, mk.sks, j.nama_jurusan, d.nama_dosen
FROM mata_kuliah mk
JOIN jurusan j ON mk.id_jurusan = j.id_jurusan
JOIN dosen d ON mk.id_dosen = d.id_dosen
ORDER BY j.nama_jurusan, mk.kode_mk;
Penjelasan: Mata kuliah punya dosen pengampu. JOIN membantu menampilkan nama dosen, bukan cuma id.
9) JOIN (laporan KRS semester 1)
JOIN WHERE ORDER BY 
SELECT m.nim, m.nama, mk.kode_mk, mk.nama_mk, k.semester, k.nilai
FROM krs k
JOIN mahasiswa m ON k.id_mahasiswa = m.id_mahasiswa
JOIN mata_kuliah mk ON k.id_mk = mk.id_mk
WHERE k.semester = 1
ORDER BY m.nama, mk.kode_mk;
Penjelasan: KRS itu “catatan pengambilan MK”. Maka harus disambungkan ke mahasiswa dan MK.
10) JOIN lanjutan (KRS + dosen pengampu)
JOIN WHERE ORDER BY 
SELECT m.nim, m.nama, mk.kode_mk, mk.nama_mk, d.nama_dosen, k.semester, k.nilai
FROM krs k
JOIN mahasiswa m ON k.id_mahasiswa = m.id_mahasiswa
JOIN mata_kuliah mk ON k.id_mk = mk.id_mk
JOIN dosen d ON mk.id_dosen = d.id_dosen
WHERE k.semester = 3
ORDER BY m.nama, mk.kode_mk;
Penjelasan: Ini versi “lengkap” laporan KRS: mahasiswa + MK + dosen + nilai.
Analogi: Seperti KHS: siapa mahasiswa, mata kuliah apa, siapa dosennya, nilai berapa.
Latihan cepat (DB Kuliah) 3 menit
  1. Tampilkan mahasiswa jurusan Sistem Informasi (id_jurusan = 2) angkatan 2021.
  2. Tampilkan mata kuliah dengan SKS ≥ 3 beserta dosennya.
  3. Tampilkan KRS semester 1 beserta nama mahasiswa, nama MK, dan dosen pengampu.
Petunjuk: WHERE + AND, JOIN ke dosen lewat mata_kuliah.
